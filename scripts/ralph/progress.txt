## Codebase Patterns
- Root-level CLI flags shared by `whale`, `whale exec`, and `whale resume` should be declared in `internal/ui/cli/cmd/root.go` via `bindPersistentFlags`.
- CLI startup config flows through `prepareCLIConfig`, which calls `app.LoadAndApplyConfig`; one-time command-line overrides should hook into that shared path instead of being re-parsed in subcommands.
- For CLI flags that may be explicitly passed with a default-valued result such as `--thinking=false`, re-apply the override after `LoadAndApplyConfig` using `flagChanged(cmd, name)` so config-file values do not win accidentally.
- When a public CLI flag should reject legacy aliases, validate the explicit raw input in `prepareCLIConfig` against both the normalized value and the public supported list so values like `low` or `xhigh` are rejected instead of silently remapped.
- Resuming an existing session should keep persisted session meta untouched until a new turn is written; startup-time CLI overrides belong in runtime state, not in `*.meta.json`.
- The Go test suite can be affected by ambient `DEEPSEEK_API_KEY`; unset it when running `make test` if credential fallback tests fail unexpectedly.
